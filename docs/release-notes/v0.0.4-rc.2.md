## Matika v0.0.4-rc.2

**Release candidate.** This is the second release candidate (`rc.2`) for Matika
v0.0.4, part of the **ManoMatika v0.0.1** product cut
(matika v0.0.4 + eyerate v0.0.4 + ahimsa v0.0.1). It is published as a GitHub
*prerelease* for QA validation — it is not a blessed product release. The
ManoMatika product is cut by `manomatika/manomatika` only after the QA gate
passes against the ahimsa-built installer; matika ships no installer of its own.

The version **core** is `0.0.4`; `-rc.2` is the pre-release marker on the
`VERSION` string, this tag, and this release title only. The bare core is what
all propagated files and compatibility checks use.

### Fixed
- **Native app icons (.icns / .ico) — PyInstaller freeze fix.** The PyInstaller
  build was blocked because `matika.spec` referenced `matika_icon_128.png` as the
  app icon and PyInstaller 6.21 requires Pillow to use a PNG as an icon source.
  This release ships real platform-native icons generated from the new 512×512
  square source (`matika_icon_512.png`):
  - `matika.icns` — macOS icon set (16 px through 512 px; all 9 of 10 slots are
    clean downscales; the 1024 @2x slot is a 2× upscale, acceptable for
    production).
  - `matika.ico` — Windows multi-size icon (16 / 32 / 48 / 64 / 128 / 256 px;
    all slots are clean downscales — crisp on every Windows DPI setting).
  - `matika.spec` updated with a per-platform `_ICON` selector (`sys.platform`
    at spec-parse time); Pillow is not needed and has not been added to any
    requirements file.
  - `matika_icon_128.png` is kept intact — it is an in-app web UI asset bundled
    via the existing `datas` entry and referenced by the running web UI.

### Added
- `src/matika/static/img/matika_icon_256.png` — new 256×256 square PNG icon
  source (committed alongside the 512×512 source).
- `src/matika/static/img/matika_icon_512.png` — new 512×512 square PNG icon
  master; used as the generation source for `.icns` and `.ico`.
