## Matika v0.0.4-rc.1

**Release candidate.** This is the first release candidate (`rc.1`) for Matika
v0.0.4, part of the **ManoMatika v0.0.1** product cut
(matika v0.0.4 + eyerate v0.0.4 + ahimsa v0.0.1). It is published as a GitHub
*prerelease* for QA validation â€” it is not a blessed product release. The
ManoMatika product is cut by `manomatika/manomatika` only after the QA gate
passes against the ahimsa-built installer; matika ships no installer of its own.

The version **core** is `0.0.4`; `-rc.1` is the pre-release marker on the
`VERSION` string, this tag, and this release title only. The bare core is what
all propagated files and compatibility checks use.

### Added
- **Release pipeline.** `VERSION` is the single source of truth for version
  metadata; `scripts/release.py` writes the target and propagates the bare core;
  `scripts/sync_version.py` propagates into `pyproject.toml` / `package.json` and
  adds a read-only `--check` drift mode (exit 0 clean / 1 drift / 2 config) with
  structured `--json` output.
- **Canonical SemVer authority.** Strict SemVer 2.0.0 parser
  (`_parse_semver` + `version_core()` + `is_prerelease()`) in
  `src/matika/core/paths.py`, mirrored verbatim in `scripts/sync_version.py` and
  `matika.spec`. Fail-loud on invalid input; no `"unknown"` sentinel. Establishes
  the core/suffix contract and the ladder `X.Y.Z-dev < X.Y.Z-rc.N < X.Y.Z`.
- **npm package.** `@manomatika/matika-frontend` published to GitHub Packages; a
  runtime import map resolves the bare specifier to matika's served JS. The
  `publish-npm.yml` workflow is gated to final bare-SemVer tags only â€” rc/dev
  tags never write to the registry.
- **Notes-only release job.** Tag-triggered `release.yml` creates a GitHub
  Release whose body is read from `docs/release-notes/<tag>.md`; tags carrying a
  pre-release suffix are flagged `--prerelease`.
- **Frozen-app packaging.** `matika.spec` + `launcher.py` for a plugin-agnostic
  PyInstaller build.
- **Compiled-asset guardrail.** Husky pre-commit hook + CI staleness check keep
  committed TypeScript output in sync with source.

### Changed
- Unified the core menu files onto the consolidated `*_menus.json` schema (v1.0).
- Framework-agnostic `option_sources` dispatch in
  `maintenance_activity_base.html`.
- Lowercased the `~/matika` app-data directory and `matika.log` filenames.
- Bumped `actions/checkout` and `actions/setup-node` to v6.
- Removed matika's own `RELEASES.md`; the ecosystem-wide audit log now lives in
  `manomatika/manomatika` (generated from `release-log.yaml`).
- Refreshed CLAUDE.md and developer docs for the 3-repo ManoMatika product
  architecture, the working-style discipline, and the version ladder.
